1.1 서비스 개요

리멤버는 (주)리멤버앤컴퍼니가 운영하는 명함관리 기반 직장인 플랫폼으로, 2013년 명함앱으로 출발해 약 500만 명이 쓰는 실명 기반 프로페셔널 네트워크로 확장했다. 실명·검증 경력 데이터 기반 역방향 채용(기업이 비구직 인재에게 먼저 스카웃)이 수익의 핵심이며, 2025년 글로벌 PEF EQT에 약 5,000억 원에 인수됐다.

Part 2핵심 문제 진단

표면적 불만(광고·자동 소셜화·채용 위장·가십)은 하나의 구조적 원인으로 수렴한다: "수익화가 자발적 갱신의 전제를 침식한 것." 자발적 갱신은 사용자가 앱을 '나를 위한 도구·공간'으로 느낄 때만 일어나는데, 수익화가 사용자를 '수익화 대상'으로 재정의하면서 그 전제가 흔들리고 있다.

데이터 출처 & 방법론

  • 개인 사용자: 구글 플레이스토어 + 앱스토어 리뷰 7,900건(구글 5,000 + 애플 2,900, 2014~2026.6) 직접 집계
  • B2B: 채용솔루션 전용 리뷰 채널이 없어 브런치·리멤버 커뮤니티·블라인드 후기글 수집 → 정성 참고
  • 처리: Python(pandas) + 키워드 사전 태깅, 부정 리뷰 = 평점 2 이하(n=654), 연도별 언급률 집계
  • 편향 통제: 리뷰어 = 극단 감정 사용자(셀렉션 바이어스) → '방향성'으로 해석

2.2 양쪽이 겪는 문제

개인 사용자 공급자

  • 진입 시 강제 자동재생 광고, 주요 화면 광고 노출
  • 명함 저장 → 자동 팔로우·스팸. '인맥 관리'가 '마케팅 채널 구독'이 됨
  • 커뮤니티 상단에 익명 가십(연봉·치정·사내 정치) 노출 → '비즈니스 네트워크' 정체성 희석
  • '1억 배지' = 지위 '과시' 형식 → 위화감·자괴감
  • AI 추천을 채용 제안처럼 위장, 헤드헌터 고스팅

기업·헤드헌터 수요자

  • 본질 가치: 데이터 신선도 = 상품 가치. 타 플랫폼과 중복 없는 독점 인재풀
  • 후보 접근 제한: 승인 전 연락처 비공개, 후보 고스팅 빈번 → 서칭 효율 저하
  • '제안' 이원화: '공고 일괄 발송(홍보성)' vs '개별 진짜 제안'. 통과율 차이를 공식 계정도 인정
  • AI 추천 위장: '포지션 제안'이 실제 스카웃이 아닌 AI 지원 유도인 경우 다수 → 구직자 착각 후 즉시 불합격 → 기업 브랜드 신뢰까지 훼손
양면 신뢰 훼손: 일괄 발송·AI 위장은 구직자 신뢰↓ + 채용담당자 필터링 공수↑ + 기업 브랜드 신뢰↓로 수요·공급 양면을 동시에 갉아먹는다 → 매출 70~75%인 채용 상품의 '품질 인식' 자체가 흔들림.

왜 스카웃 라벨 가시화가 1위인가

  • User Impact 최고점 (5): 부정 리뷰 원인 1위 (채용 언급률 2%→15%, '24년 3%→26% 급등)
  • Competitive Moat 강함 (4): 리멤버의 재직자 프로필+명함 관계 그래프 활용도 최고
  • Feasibility 높음 (4): 기존 스카웃 파이프 재설계, 3~4개월 개발 가능
  • Revenue Impact 중상 (4): B2B 매출 방어, 개인화 아웃리치 답장률 3배 벤치마크
결정적 논리: 다른 개선안(광고 통제·시장 리포트 등)은 이걸 안 하면 무의미. 스카웃 신뢰가 회복 안 되면 사용자는 어차피 이탈 → 다른 처방은 자산 파괴된 이후의 사후약방문. 스카웃 재설계가 다른 모든 처방의 전제 조건.

이 포트폴리오의 스코프: 5개 중 1순위 스카웃 라벨 가시화만 심층 구체화. 나머지 4개는 Phase 2·3에서 순차 진행 (별도 프로젝트).

4.3 스카웃 라벨 가시화 심층

프레임워크: Customer Journey Map + Behavioral Psychology + Peak-End Rule + Implicit Contract Theory, 데이터: 실사용 관찰, 스크린샷, 리뷰 7,900건, 심리학 이론

사용자 여정: 7단계 감정 곡선

User Journey Map: 7단계 감정 곡선 (HTML 재구성). '끝'이 무응답이면 전체 기억이 부정으로 지배됨
#단계감정심리 원리
1알림 수신호기심·기대Reciprocity (Cialdini)
2상세 진입혼란·불안Choice Paralysis, Loss Aversion
3결정 시도부담·회피Commitment Escalation 부재
4무행동 (Freeze)방치Status Quo Bias
5"물어보세요" 안내 따라 질문다시 시도Implicit Contract
6무응답 (Ghosted)좌절·굴욕Learned Helplessness, Peak-End
7다음 스카웃 도착냉소·무관심Confirmation Bias

심리 진단 3가지

  • Choice Paralysis (단계 2-3): 극단 이분법이 결정 마비 → 무행동
  • Broken Implicit Contract (단계 5-6): 회사가 먼저 두드린 계약을 무응답으로 파기
  • Peak-End Rule 실패 (전체): 경험의 "끝"이 무응답이면 전체 기억 부정

스카웃의 심리적 비대칭성

Part 7회고

이 프로젝트를 통해 얻은 배움들.

1. 프레임은 도구지 답이 아니다

처음엔 매슬로 욕구 위계로 사용자 가치를 정리했다. 그럴듯해 보였지만 결국 "리멤버는 존중 욕구를 자극한다"는 서사에 사용자 경험을 끼워 맞추게 됐다. 편향이었다. JTBD와 Value Proposition Canvas로 바꾸고 실사용자 후기·리뷰 데이터로 채우니 훨씬 정확한 진단이 나왔다. 프레임을 먼저 정하고 데이터를 끼워 맞추는 게 아니라, 질문에 맞는 프레임을 골라야 한다.

2. 정량 데이터로는 안 보이는 페인이 있다

리뷰 7,900건과 시장 벤치마크로도 잘 드러나지 않은 UX 페인이, 실제로 앱을 써보면서 발견됐다. 스카웃 상세 페이지에서 "수락"과 "다음 기회에" 앞에서 결정을 못한 그 순간이 그것이다. 무응답 방치 경험은 "고백 안 했는데 차인 느낌"이라는 표현으로 정리됐다. 정량 데이터는 문제의 규모를 보여주지만 감정과 결(質)까지 담기는 어렵다. 리서치를 보완하는 관점으로 직접 사용해보는 과정이 함께 필요하다.

3. 가설을 팩트로 착각하지 않을 용기

처음엔 "리멤버 사용자가 이탈 중"이라고 단정했다. 근데 회원 수는 오히려 성장 중이었다. 이탈의 직접 증거는 없었다. 정직하게 "silent decline"이라는 가설로 재정의하고 검증 계획을 붙이니 오히려 진단이 강해졌다. 확신을 자제하고 "이건 가설이다"라고 말할 수 있어야 진짜 PM이다.

4. 반박이 진짜 발견을 만든다

콜드콜 시스템화 → "접근 마찰이 없어지면 스팸이 된다" 반박 → 폐기. 평점 하락 서사 → "그건 4.8이잖아" 반박 → Silent decline이라는 더 정교한 진단. 매슬로 프레임 → "너무 편향적" 반박 → JTBD 전환. 콜드콜 재도전 → "고백 안 했는데 차인 느낌" 통찰. 스스로에게 반박할 수 있어야 한다. 확신은 발견을 막는다.

5. 스코프 규율이 완성도를 만든다

처음엔 5개 개선안을 다 구체화하려 했다. 헤드헌터·기업 표본 부족을 인정하고 개인 사용자만 남겼고, 다시 좁혀 스카웃 라벨 가시화 하나만 심층으로 갔다. 역설적으로 좁힐수록 서사가 강해졌다. 다 하려다 아무것도 안 되는 것보다, 하나를 깊이 파는 게 낫다.

6. AI는 발산, 인간은 수렴

6명 전문가 페르소나를 AI 에이전트로 병렬 실행해 42개 아이디어를 확보했다. 혼자서는 못 낼 다양성이었다. 하지만 어피니티 클러스터링·결정 매트릭스·최종 판단은 결국 인간의 일이었다. AI는 발산(다양성 확보)에 강하고, 판단(수렴)은 인간이 해야 한다.

7. 표면 지표는 속인다

앱스토어 총점 4.8만 보면 리멤버는 문제 없는 앱이다. 그런데 최근 연도별 리뷰 평균은 2.56이고, 리뷰 수 자체가 40배 감소했다. 초기 대량 긍정이 총점을 유지시켜 문제를 감추고 있었다. lagging 지표(누적 평점)와 leading 지표(최근 리뷰 감정)의 갭이 가장 위험한 신호다.

리멤버는 훌륭한 서비스다. 이 프로젝트는 "리멤버가 틀렸다"가 아니라 "옳은 방향으로 확장하려다 실행에서 균형을 잃었다"는 진단이다. Product를 훼손하지 않으면서 Platform으로 확장하기: 이 균형을 잡는 순간 리멤버는 시장 순풍을 온전히 활용할 수 있다. 그 첫 걸음이 스카웃 라벨 가시화다.
